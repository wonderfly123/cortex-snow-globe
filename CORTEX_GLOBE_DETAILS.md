# Cortex Snow Globe

Interactive 3D globe visualization for global food truck analytics, powered by Snowflake and Cortex AI. Displays 30 cities across 15 countries with real-time KPIs, charts, and AI-generated business insights.

## Tech Stack

- **Frontend:** Next.js 16, React 19, React Three Fiber, Three.js, Recharts, Framer Motion, Tailwind CSS 4, Zustand
- **Backend:** Next.js API routes, Snowflake SDK
- **AI:** Snowflake Cortex (`mistral-large2` via `SNOWFLAKE.CORTEX.COMPLETE`)
- **Database:** Snowflake (`TAKEHOME_DB` — `HARMONIZED.POS_FLATTENED_V`, `ANALYTICS.*` views/tables)

## Architecture

```
page.tsx
├── Globe (React Three Fiber canvas)
│   ├── Earth (textured sphere)
│   ├── CityMarkers (interactive sprites, sized by sales)
│   ├── Atmosphere (custom shader halo)
│   └── CameraController (fly-to animation on city select)
├── CityCard (left panel — KPIs, sparkline, top items, AI insight)
├── GlobalKPIBar (top-right — revenue, orders, AOV + time slider)
└── AnalyticsPanel (bottom drawer — 6 tabs with charts + AI insights)
```

## Features

### 3D Interactive Globe
- Auto-rotating Earth with bump-mapped terrain and atmospheric glow
- City markers scaled by total sales, color-coded by state (default/hover/selected)
- Click a city: camera flies to location, loads detail panel
- Bloom + vignette post-processing, 1500-star background

### City Detail Panel (CityCard)
When a city is selected, a left-side panel shows:
- **KPI grid:** Revenue, Orders, Avg Order Value, Active Trucks
- **Sparkline:** Monthly sales trend (SVG with gradient fill)
- **Top 3 items** by revenue
- **AI narrative** generated by Cortex Complete

### Global KPI Bar
- Aggregated Revenue, Orders, and Avg Order Value across all cities
- **Time range slider** filtering from 2021-01 to 2022-11 (dual-handle)
- KPIs update dynamically based on selected range

### Analytics Panel (Bottom Drawer)
Draggable bottom sheet with 6 tabs, each with charts and an AI insight:

| Tab | Visualization | Data Source |
|-----|--------------|-------------|
| Sales Trend | Area chart (all countries) + multi-line by country with interactive legend | `MONTHLY_TREND_DT` |
| Top Brands | Horizontal bar charts per year | `POS_FLATTENED_V` |
| Menu Types | Horizontal bar chart with country selector | `POS_FLATTENED_V` |
| Franchisees | Card grid with country/year selectors | `POS_FLATTENED_V` |
| Distribution | Custom SVG donut + vertical bar chart | `CITY_KPI_DT` |
| Patterns | 3-column grid: monthly area, day-of-week bars, hourly bars | `MONTHLY_TREND_DT` + `POS_FLATTENED_V` |

Each tab includes a **Cortex AI Insight** — a 3-4 sentence analysis generated from the underlying query data.

## Cortex AI Integration

Two API endpoints call `SNOWFLAKE.CORTEX.COMPLETE('mistral-large2', prompt)`:

### 1. City Narrative (`/api/cortex/complete`)

Generates a 2-3 sentence insight for a selected city.

```typescript
// Fetch city KPIs and top item, then call Cortex
const prompt = `You are a food truck business analyst. Analyze this city's performance...
City: ${kpi.CITY}, ${kpi.COUNTRY}
Total Sales: ${formatCurrency(kpi.TOTAL_SALES)}
Total Orders: ${kpi.TOTAL_ORDERS.toLocaleString()}
Average Order Value: ${formatCurrency(kpi.AVG_ORDER_VALUE)}
Top Selling Item: ${topItem}
Provide a concise, data-driven insight:`

const result = await executeQuery<CortexResult>(`
  SELECT SNOWFLAKE.CORTEX.COMPLETE('mistral-large2', ?) AS AI_NARRATIVE
`, [prompt])
```

### 2. Tab Analysis (`/api/cortex/tab-insight`)

Generates a 3-4 sentence analysis for each analytics tab. Runs tab-specific SQL queries, formats results as a pipe-delimited table, and sends to Cortex with a business question.

```typescript
// Example: sales-trend tab runs two queries
const queries = [
  `SELECT MONTH, SUM(SALES) AS TOTAL_SALES, SUM(ORDERS) AS TOTAL_ORDERS
   FROM TAKEHOME_DB.ANALYTICS.MONTHLY_TREND_DT
   GROUP BY MONTH ORDER BY MONTH`,
  `SELECT COUNTRY, SUM(SALES) AS TOTAL_SALES, SUM(ORDERS) AS TOTAL_ORDERS
   FROM TAKEHOME_DB.ANALYTICS.MONTHLY_TREND_DT
   GROUP BY COUNTRY ORDER BY TOTAL_SALES DESC`,
]

// Format results as table, build prompt
const prompt = `You are a senior business analyst for a global food truck company.
Question: What is the overall trend in sales across all countries?...
DATA:
${dataStr}
Provide a 3-4 sentence insight...`

const result = await executeQuery<CortexResult>(`
  SELECT SNOWFLAKE.CORTEX.COMPLETE('mistral-large2', ?) AS AI_NARRATIVE
`, [prompt])
```

Both endpoints use **1-hour in-memory caching** to avoid redundant Cortex calls.

## Snowflake Connection

Supports three auth methods (checked in order):

```typescript
// 1. Inline private key (Vercel deployment)
if (process.env.SNOWFLAKE_PRIVATE_KEY) {
  config.privateKey = process.env.SNOWFLAKE_PRIVATE_KEY.replace(/\\n/g, '\n')
  config.authenticator = 'SNOWFLAKE_JWT'
}
// 2. File-based private key (local dev)
else if (process.env.SNOWFLAKE_PRIVATE_KEY_PATH) {
  config.privateKey = fs.readFileSync(keyPath, 'utf-8')
  config.authenticator = 'SNOWFLAKE_JWT'
}
// 3. Password auth
else if (process.env.SNOWFLAKE_PASSWORD) {
  config.password = process.env.SNOWFLAKE_PASSWORD
}
```

Connection is pooled as a singleton — one connection shared across all API route invocations.

## State Management (Zustand)

Single store manages all application state:

```typescript
// Key state slices
cities: CityData[]              // Globe markers
selectedCity / selectedCountry  // Current selection
cityKPI / monthlyTrend / topItems / narrative  // City detail data
analyticsTab / timeRange        // Analytics panel UI
salesTrendData / topBrandsData / ...  // Cached analytics data (null = not fetched)
tabInsights: Record<string, string>   // Cached AI insights per tab
```

Analytics data uses a **lazy-fetch pattern** via `useAnalyticsData` hook — data is `null` until the tab is first viewed, then cached in the store.

## API Routes

| Route | Method | Source Table | Purpose |
|-------|--------|-------------|---------|
| `/api/cities` | GET | `CITY_COORDINATES` + `CITY_KPI_DT` | Globe markers with sales data |
| `/api/city` | GET | `CITY_KPI_DT` + `MONTHLY_TREND_DT` + `TOP_ITEMS_DT` | City detail panel data |
| `/api/analytics/sales-trend` | GET | `MONTHLY_TREND_DT` | Monthly sales by country |
| `/api/analytics/top-brands` | GET | `POS_FLATTENED_V` | Top 3 brands per year |
| `/api/analytics/menu-types` | GET | `POS_FLATTENED_V` | Top 3 cuisines per country |
| `/api/analytics/franchisee-months` | GET | `POS_FLATTENED_V` | Best months per franchisee |
| `/api/analytics/order-distribution` | GET | `CITY_KPI_DT` | Orders/sales by country |
| `/api/analytics/order-patterns` | GET | `MONTHLY_TREND_DT` + `POS_FLATTENED_V` | Monthly, DOW, hourly patterns |
| `/api/cortex/complete` | POST | `CITY_KPI_DT` + `TOP_ITEMS_DT` + Cortex | City AI narrative |
| `/api/cortex/tab-insight` | POST | Various + Cortex | Per-tab AI analysis |
